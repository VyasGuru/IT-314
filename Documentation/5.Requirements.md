# Real Estate Listing Website – Requirements Specification

---

## Elicitation Techniques Used

### Questionnaires / Surveys
We reached out to a wide range of potential users through online forms and surveys. Their responses gave us a clearer picture of the features they actually care about and how they expect the website to function.

Justification: "It highlights the things users care about the most i.e. as saving their favourite properties or comparing prices-and helps us decide which features should be prioritised because they’ll actually be useful to people

### Brainstorming
We held open discussions with our team members to get as many ideas as possible for the website. Everyone was able to contribute freely without worrying about whether the idea was perfect or not.
Justification: This type of approach helps us to discover creative features & solutions that we might not think of otherwise and gives us a pool of ideas to refine into real requirements.

### Use Cases & Scenarios
We learned about various actions of all kinds of users as an example: users want to save the listing or request a visit for a property. We also considered things which have chances to go wrong  like a documentis expired or a request for a property visit  being rejected .
Justification: This makes sure that we don’t miss any important details and also how the system should work in all types of cases.

### Analysis of Existing Systems
By looking at popular real-estate websites that are already outthere like Bayut and99acrest see how they work and what features they offer. This helped us a lot in understanding how this type of website is working.
Step by step, we mapped the typical actions of different users — adding a property, saving listings, requesting a visit. We also thought about what could go wrong, such as a document that is expired or a request that gets rejected.
Justification:  this ensure we won’t overlook something important and it gives us an idea of how our system should behave in all cases, not just the good ones.

### Analysis of Existing Systems
We looked at a few of the popular real estate sites, like Bayut and 99 acres to understand how they work and what features do they offer. This was useful for us to learn a lot about what users are expecting and what's the norm in this industry.

Justification:   We need to learn from our predecessors’ systems what features we have to add / what errors to avoid, in order that our system will be effective and competitive.

## 2. Functional Requirements

### 2.2 Registered User (Buyer / Renter)

#### Must Have
Buyers/renters should be able to securely log in on the platform.  
Buyers/renters will need to upload valid proof of their identification in the system.  
Buyers/renters should be able to see price comparisons for different properties.  
Buyers/renters can save property as well for reference.  
Buyer/Renters should be able to search their properties according to desired property using filter.

#### Should Have
- The system shall allow listers to view feedback and reviews related to their properties.
- The system shall allow listers to view requests from buyers/renters.

#### Could Have
- The system shall provide a 360° property view feature.
- The system shall allow buyers/renters to set alerts/notifications for new properties matching their criteria.

Justification  
- Secure login & verification:  This Ensureses authenticity and prevents fraud.  
- Add/update/delete listings: Core/important functionality for property management.  
- View feedback & requests: Supports communication and service quality.
- Hiding feature: temperoriy hides without deleting it will ensure flexibility and convenience.

### 2.3 Registered User (Lister / Seller / Owner / Real Estate Agent)

#### Must Have

- The system shall require listers to upload valid identity proof.
- The system shall allow listers to add, update, and delete property listings.
- The system shall allow lister to temperorily hide their listed property.

Listers must be able to see what feedback and reviews they have concerning their properties.  
The system should permit listers to be able to take a look at the requests from buyers/renters.

Justification  
Secure login & verification: Provides secure authentication and helps prevent fraud.  
Create/update/delete listing: Basic features for property management.  
Read messages & requests: Improved communication and level of service.  
hiding feature:temperoriy hide without removing it, more flexible and simple.

## 3. Non-Functional Requirements

- Performance :The system shall react to the user requests in a few seconds even with an average heavy workload.  
- Scale: The system must scale to at least 10,000 simultaneous users.  
- Security: All passwords shall be stored in encoded format by the system.  
- Usability Requirement The following requirement is essential for a usable system: R Defence – A user friendly interface should be provided by the system.  
- Uptime The service must be available for most users throughout the year.  
- Compatible The system shall be accessed via desktop and mobile browsers.  
- Maintenance The system must support easy updating of property categories and features so that there is no or limited downtime.

Justification  
Performance & scalability: smooth experience and future growth.  
Security: Protects sensitive data.  
Usability & reliability: Builds trust, satisfaction.  
Device friendly: Compatible of all the devices.  
Maintainability: Supports continuous improvement.

## 4. Domain Requirements

### Must Have
The application will be designed to adhere with real estate laws and regulations of the country in which it will operate (for example, RERA compliance – India).  
The system will ensure that only valid property documents (owner title, legal clearances) are uploaded and stored in it.  
The system will be able to manage different kinds of properties: residential properties real estates for rent,commercial properties and land.  
-Listing history will be kept by the system for legal and audit reasons.  
The data confidentiality shall be guaranteed by the system.

### Could Have
The system will keep up with property location information with the help of maps.

Justification  
Legal Compliance & verified docs: Keeps fraud at bay and builds trust.  
Various types of attributes & maps: Provides flexibility.  
Data privacy: Safeguards privacy by protecting sensitive data.
